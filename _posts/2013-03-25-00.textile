h3. Альтернативные прошивки для Ubiquiti M-серии (и клонов)

p. **Для создания прошивок использовался SDK от Ubiquiti версии 5.5.0.**

p. **Изменения и дополнительные программы в прошивках :**

* по-умолчанию устанавливается страна как "Compliance Test" в которой открыт весь диапазон с шагом 5 MHz и сняты ограничения по выходной мощности передатчика
* отключена функция автоматической проверки обновлений версий прошивок на сайте Ubiquiti
* при обновлении через WEB-интерфейс не ругается (warning) о том, что прошивка альтернативная
* в ядро добавлена поддержка tun/tap (L2/L3 туннели openvpn и vtun)
* в bysybox добавлены аплеты - awk, head, tr, wc, wget
* при заходе по SSH в консоли отображается имя хоста, а не версия прошивки
* добавлена команда fs (алиас), позволяющая оперативно сохранить данные (скрипты и т.д.), находящиеся в каталоге /etc/persistent/
* добавлен набор скриптов для отправки статуса  устройства на сервер FlyMon (будет удален в следующих версиях в связи с переходом на другой протокол обмена данными)
* добавлена утилита httping для проверки доступности ресурсов по протоколу HTTP
* добавлена возможность создания легких и быстрых L2/L3 туннелей на базе vtun
* добавлен zabbix_agentd для возможности отправки статуса и параметров устройства на сервера Zabbix и FlyMon (устанавливается пользователем)
* добавлен mjpeg-streamer для подключения USB UVC WEB-камеры с возможностью удаленного просмотра (пока поддерживаются устройства только на процессоре 7241)
* добавлена утилита fixsum и скрипт pistacho для горячей смены MAC адреса прямо на устройстве, подробнее можно прочитать "тут":http://blog.flyrouter.net/2012/12/mac-hwid-eeprom-ubiquiti.html

p. Подробнее о аппаратной переделке TP-Link WA7510N в Ubiquiti можно прочитать "тут":/pages/2013051100.html

p{color:pink}. У переделанного в Ubiquiti TP-Link WA7510N на данный момент не работает индикация уровня сигнала и индикация активности Ethernet порта. Проблема решена в новой версии прошивки, релиз которой будет в августе-сентябре.

h3. Прошивки для TP-Link WA5210G, переделанных в Ubiquiti

p. **Для создания прошивок использовался SDK от Ubiquiti версии 4.0.2.**

p. **Изменения и дополнительные программы в прошивках :**

* по-умолчанию устанавливается страна как "Compliance Test" в которой открыт весть диапазон с шагом 5 MHz и сняты ограничения по выходной мощности передатчика
* при обновлении через WEB-интерфейс не ругается (warning) о том, что прошивка альтернативная
* исправлена индикация уровня сигнала (TP-Link использует другие GPIO)
* в ядро добавлена поддержка tun/tap (L2/L3 туннели openvpn и vtun)
* добавлен модуль proc_gpio.o и простой скрипт контроля gpioctl
* в bysybox добавлены аплеты - awk, head, tr, wc, wget
* добавлена утилита hexdump для просмотра раздела EEPROM
* при заходе по SSH отображается имя хоста, а не версия прошивки
* добавлен набор скриптов (вкладка Services) для отправки статуса  устройства на сервер FlyMon (будет удален в следующих версиях в связи с переходом на другой протокол обмена данными)
* заменен логотип устройства (спасибо kuslik с форума "www.lan23.ru":http://www.lan23.ru)

p{color:pink}. У переделанного в Ubiquiti TP-Link WA5210G на данный момент не работает антенна с горизонтальной поляризацией (вертикальная и внешняя - работают). Проблема решена в новой версии прошивки, релиз которой будет в августе-сентябре.
